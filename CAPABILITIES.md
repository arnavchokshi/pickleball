# Racket-Sport Capability Truth Matrix

Last audited: 2026-06-30.

This matrix records what the current pipeline actually invokes. `BUILD_CHECKLIST.md` is the operational source for mutable task counts and current row status totals. A smoke, probe, precomputed artifact adapter, CPU scaffold, or presence-count check is not `DONE` for a row that names a specific model or algorithm, and it never counts as `VERIFIED`.

## Canonical Tier Split

This section is the single source of truth for tier placement. Other docs may
reference it, but must not invent a different live/server split.

**ON-DEVICE LIVE / fast tier (<10s, runs on iPhone ANE during play):**

- Court calibration: once at setup, cached; seed from ARKit intrinsics+pose+floor plane.
- Person detect + track + N-lock: CoreML YOLO26n/s + lightweight ByteTrack/BoT-SORT + court-polygon filter.
- 2D pose/joints: Apple Vision or CoreML RTMPose-m.
- Ball tracking: distilled/quantized CoreML heatmap tracker (~288p). MAIN on-device optimization RISK.
- Ball IN/OUT + net-crossing line calls: homography + bounce on cached court; known Z=0 resolves single-cam depth.
- Contact/hit timing: on-device mic onset + wrist-velocity peak.
- Live court map + 1 priority cue + capture-quality guidance.

**SERVER OFFLINE / deep tier (async, GPU):**

- 3D mesh: Fast SAM-3D-Body / SAT-HMR.
- World grounding: WHAM/TRAM/GVHMR/OnlineHMR plus court-plane hard constraint.
- Foot-lock + physics.
- Paddle 6DoF.
- Full biomech.
- 3D replay render.
- LLM coaching copy.
- Week-over-week.

**BORDERLINE:**

- Camera-space mesh preview = `server-fast`, not phone-real-time.
- LiDAR = near-field (~5m) bonus only.

## Gate Labels

Current evaluator gates for body, foot/physics, metrics, replay, and e2e are presence, artifact-completeness, or structural artifact checks. Replay/e2e GLB checks parse GLB 2.0 header/chunk/glTF JSON structure, but they do not prove visual correctness, animation quality, compression, USDZ export, or accuracy. Tracking, ball/events, and racket now add `label_check.*` metrics only when reviewed labels are authoritative; prototype draft or `not_ground_truth` labels are reported as `not_measured`. Racket artifact player/frame/contact counts are recorded as `artifact_check.*` context and cannot pass Phase 6 without reference pose labels. Presence checks, artifact checks, sparse prototype benchmarks, and unreviewed label checks do not count toward `VERIFIED`.

Shot classification gates now report reviewed-label macro-F1, top-2 accuracy, missing-prediction coverage, per-class precision/recall/F1, unknown/gated rates, and confusion when authoritative labels exist. The current calibration bucket helper lives in `shot_event_eval.py`, but the phase gate does not yet surface a dedicated calibration metric. Current external checks are insufficient for promotion: THETIS 60-clip family accuracy is 51.7% overall / 66.7% top-2 with `serve` 0/15 and `overhead` 0/5, while OpenSportsLab 100-event broad accuracy is 81.0% but `serve` is 0/19. Broad swing or FH/BH success cannot be used to claim serve, overhead, or full shot-taxonomy readiness.
