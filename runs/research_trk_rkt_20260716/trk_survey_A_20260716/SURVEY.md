# TRK survey lane A — detector, segmentation, domain adaptation, and ReID

Date: 2026-07-16  
Scope: independent lane A; no sibling lane material was read.  
Status: research-only. **VERIFIED=0.** Published results below justify experiments; none establish DinkVision accuracy or promotion.

## Evidence and license conventions

- **[FETCHED-PRIMARY]** means the linked paper, official repository, repository license, model card, or release listing was fetched in this survey.
- **[SECONDARY]** means a non-author source. No load-bearing recommendation below relies on one.
- **[INFERENCE]** means an engineering judgment for the stated DinkVision setting, not a published result.
- License posture is deliberately end-to-end: `commercial-clean` requires a commercially usable path for code, weights, and training data; `R&D-only` has an explicit noncommercial/academic restriction; `unknown-needs-review` has a missing or custom term. An Apache/MIT repository does not cleanse restricted checkpoint training data.
- Runtime numbers are the authors' model-only measurements on the named hardware. They are not end-to-end 1080p60 pipeline measurements.

## Executive answer

1. **[INFERENCE] RF-DETR-L detection remains the first zero-shot challenger**, followed immediately by RF-DETR-Seg-L as the mask-producing A/B. Its combination of 704 input, 56.5 COCO AP, Apache-designated weights, and now-mature custom training path is the best fit for the coverage gap. There is still **no official RF-DETR person-class, CrowdHuman, or MOT20-det result**; superiority under net/crossing occlusion is unproved.
2. **[INFERENCE] Local domain supervision is more likely to move the gate than swapping among close COCO leaders.** The launch-relevant dataset must label on-court players, spectators/passers, truncated players, and empty/hard-negative frames. SportsMOT is excellent R&D pre-adaptation because it explicitly excludes non-player humans, but it is CC BY-NC 4.0.
3. **[FETCHED-PRIMARY] The only material post-register release is SAM-MT**, posted July 9–10, 2026. It reports substantially better multi-target video-mask throughput than SAM 2.1, but its checkpoint is CC BY-NC-SA 4.0, the GitHub tree has no license file, and training code is absent. It is an R&D mask-source control, not a product dependency.
4. **[INFERENCE] No surveyed public person-ReID checkpoint is launch-clean.** OSNet/Market, SOLIDER, TransReID, CLIP-ReID, KPR, and sports checkpoints inherit academic/noncommercial or unresolved dataset terms. The clean path is a permissively licensed architecture initialized from a cleared backbone and trained/fine-tuned only on consented DinkVision crops.

### Disagreements and source preference

- **[FETCHED-PRIMARY]** RF-DETR's current table gives D-FINE-L Objects365→COCO as 57.2 AP, while D-FINE's own repo/paper reports 57.3. This is rounding/benchmark-harness drift; this survey uses the author repo's 57.3 and does not treat 0.1 AP as meaningful.
- **[FETCHED-PRIMARY]** GitHub labels RF-DETR's repository Apache-2.0, while its README explicitly carves XL/2XL detection and `rfdetr_plus` into PML 1.0. The component-specific README/license statement controls; GitHub's single repository badge is not sufficient.
- **[FETCHED-PRIMARY]** HF model cards may label a checkpoint Apache/MIT while its training datasets impose research-only terms. This survey trusts the narrowest applicable primary term and flags the combined posture accordingly.

- **[FETCHED-PRIMARY] No direct public racket-sport person-MOT set was located** with boxes/IDs, a working artifact, and usable commercial terms. Existing racket datasets found focus on ball/racket/action rather than four-person MOT.
- **[FETCHED-PRIMARY] Evidence limit.** The primary sports papers show in-domain-trained detectors/trackers and practical recipes, but this survey found no controlled study matching “hundreds-to-thousands of boxes, fixed elevated four-player camera” with a clean zero-shot-versus-small-fine-tune ablation. Therefore “small local fine-tune wins” is a high-confidence **[INFERENCE]**, not quoted fact.

### Practical adaptation recipe

1. **[INFERENCE] Freeze the evaluation first.** Score each zero-shot detector on the identical worst-clip detections before any training. Split local labels by game/camera/session, never adjacent frames.
2. **[INFERENCE] Train the role the product needs.** Prefer `on_court_player` as the positive class. Include every visible/truncated/occluded player; densely include spectator/passers and far-off-court humans as explicit `non_player_person` hard negatives or a separate diagnostic class. Add empty-court and sideline-only frames. Do not leave unlabeled people in training images.
3. **[INFERENCE] Preserve geometry without making it a brittle detector gate.** Use full-scene images and rectangular resize; optionally add a court-conditioned crop branch. Keep a margin for lunges and frame truncation. Apply court-region rejection after detections as a scored product rule, not by deleting positive edge examples.
4. **[INFERENCE] Resolution A/B.** For RF-DETR-L compare native 704 square with a higher/rectangular configuration if supported; use `scale_jitter=False` in the edge-truncation branch. Score far-side recall separately.
5. **[INFERENCE] Low-data schedule.** Run (a) frozen encoder/LoRA 10–20 epochs and (b) full fine-tune 20–40 epochs from the same seed/data. Stop from held-out gate metrics, not training loss. Repeat the winning recipe with at least three seeds before considering stability.
6. **[INFERENCE] Data budget.** First tranche: 1k–2k carefully chosen boxes plus 500–1k hard-negative people/frames; second tranche only from residual false negatives/FPs. Annotations, not raw frame count, are the scarce unit.

## Recommended benchmark order

This is a specification only; no GPU dispatch is authorized or performed.

1. **RF-DETR-L zero-shot detection, pinned Apache weight.** Run at the official 704 input on the frozen worst-clip detections. Report player recall/coverage by near/far side, occlusion, and truncation; spectator/far-off-court FP separately. Keep tracker/association fixed.
2. **RF-DETR-Seg-L zero-shot as a detector+mask A/B.** First score its boxes with the same tracker and no mask cue. Then archive masks for later McByte. This separates detector gain from mask-association gain.
3. **Independent detector controls:** D-FINE-L Objects365→COCO, then DEIMv2-L; add ECDet-L only if those do not explain whether RF's gain is architecture or resolution. No threshold grid: use documented defaults plus at most one predeclared calibration from validation, not worst-clip tuning.
4. **Local detector adaptation on the best two zero-shot families.** Begin with RF-DETR-L frozen-encoder/LoRA and full fine-tune branches using the hard-negative schema above; use `scale_jitter=False` for the edge-truncation branch. Re-score the unchanged gate.
5. **Segmentation source comparison only after detector selection:** RF-Seg-L vs YOLO26m-seg coarse masks; SAM2.1/EfficientTAM re-prompt propagation only if per-frame masks are too expensive or temporally unstable.
6. **ReID:** A/B SOLIDER against frozen OSNet as an R&D diagnostic, then train the cleared session-specific embedding. Keep detections and association logic fixed.
7. **McByte mask cue on the worst clips.** Feed the selected detector and precomputed masks, preserve all non-mask association settings, and report incremental switches/coverage/FP. If McByte fails, diagnose mask drift versus association use before considering MASA.
8. **CAMELTrack only if detector/domain/ReID/mask leverage still misses the gate.** Its public global checkpoint is not product-clean and learned association violates the intended order if run earlier.
